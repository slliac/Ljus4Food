\subsection{Detailed Description}
Invocation matcher which looks for sets of specific parameters in the invocations.

Checks the parameters of the incoming invocations, the parameter list is checked against the defined constraints in \$parameters. If the constraint is met it will return true in \mbox{\hyperlink{class_p_h_p_unit___framework___mock_object___matcher___consecutive_parameters_a4988c6fa11e275302172d0a4ae32dd3a}{matches()}}.

It takes a list of match groups and and increases a call index after each invocation. So the first invocation uses the first group of constraints, the second the next and so on.
